SQL Implementation

Core Analytics Query

This comprehensive SQL query processes GA4 raw data to extract meaningful custom event insights with domain-level aggregations:

custom_events_analysis.sql 
WITH aggregated_events AS (
  SELECT
    host_name,
    event_name,
    SUM(event_count) AS event_count,
    ROUND(SAFE_DIVIDE(
      SUM(event_count_without_session_start),
      COUNT(DISTINCT user_pseudo_id)
    ), 2) AS events_per_user
  FROM
    (
      SELECT
        device.web_info.hostname AS host_name,
        event_name AS event_name,
        COUNT(event_name) AS event_count,
        COUNTIF(event_name NOT IN ('session_start')) AS event_count_without_session_start,
        user_pseudo_id
      FROM
        `analytics_project.analytics_123456789.events_*`
      WHERE
        -- Dynamic date range for last 7 days excluding today
        _TABLE_SUFFIX BETWEEN FORMAT_DATE(
          '%Y%m%d',
          DATE_SUB(CURRENT_DATE(), INTERVAL 7 DAY)
        ) AND FORMAT_DATE(
          '%Y%m%d',
          DATE_SUB(CURRENT_DATE(), INTERVAL 1 DAY)
        )
        -- Exclude automatically collected events
        AND event_name NOT IN (
          -- Automatically Collected Events
          'first_visit', 'session_start', 'page_view', 'scroll', 
          'engagement_time_msec', 'app_store_subscription_cancel', 
          'app_store_subscription_convert', 'app_store_subscription_renew',
          'app_update', 'user_engagement',
          -- Enhanced Measurement Events
          'video_engagement', 'file_download', 'search', 'scroll', 
          'form_interaction', 'click'
        )
      GROUP BY
        host_name,
        event_name,
        user_pseudo_id
    )
  GROUP BY
    host_name,
    event_name
),
domain_metrics AS (
  SELECT
    host_name,
    event_name,
    event_count,
    events_per_user,
    -- Count total unique events per domain
    COUNT(event_name) OVER(PARTITION BY host_name) AS events_per_domain,
    -- Sum total events per domain  
    SUM(event_count) OVER(PARTITION BY host_name) AS total_events_sum_domain,
    -- Calculate event share within domain
    ROUND(
      event_count / SUM(event_count) OVER(PARTITION BY host_name), 
      3
    ) AS event_share_domain
  FROM aggregated_events
)

-- Final output with comprehensive metrics
SELECT
  host_name,
  event_name,
  event_count,
  events_per_user,
  events_per_domain,
  total_events_sum_domain,
  event_share_domain,
  -- Add ranking within domain
  ROW_NUMBER() OVER(
    PARTITION BY host_name 
    ORDER BY event_count DESC
  ) AS event_rank_in_domain,
  -- Add global ranking
  ROW_NUMBER() OVER(
    ORDER BY event_count DESC
  ) AS global_event_rank
FROM domain_metrics
ORDER BY 
  event_count DESC,
  host_name ASC,
  event_name ASC;

Query Architecture Breakdown

1. Data Extraction Layer

The innermost query processes raw GA4 events table, extracting hostname from device information and applying intelligent date filtering for the last 7 days.

  • Dynamic Date Range: Uses FORMAT_DATE with DATE_SUB for rolling 7-day windows
  • Event Filtering: Excludes automatically collected and enhanced measurement events
  • User-Level Aggregation: Groups by user_pseudo_id to enable per-user calculations

2. Event Aggregation Layer

The aggregated_events CTE performs domain and event-level aggregations while calculating sophisticated metrics.

  • Event Count: Total event occurrences per domain/event combination
  • Events Per User: Average events per unique user (excluding session_start)
  • Safe Division: Uses SAFE_DIVIDE to prevent division by zero errors

3. Domain Analytics Layer

The domain_metrics CTE adds window functions for domain-level comparative analysis.

  • Domain Event Diversity: Count of unique events per domain
  • Domain Volume: Total event volume aggregated by domain
  • Event Share: Percentage contribution of each event within its domain

Technical Implementation Details

Data Source Configuration

The query targets GA4's exported events tables using dynamic table suffixes to process date-partitioned data efficiently.

`analytics_project.analytics_123456789.events_*`

Dynamic Date Filtering

Implements rolling 7-day analysis windows with automatic date calculation, excluding the current day for complete data.

_TABLE_SUFFIX BETWEEN FORMAT_DATE('%Y%m%d', DATE_SUB(CURRENT_DATE(), INTERVAL 7 DAY))

Window Functions

Advanced window functions calculate domain-level aggregations and rankings without additional query complexity.

SUM(event_count) OVER(PARTITION BY host_name) AS total_events_sum_domain

Error Handling

Uses SAFE_DIVIDE function to gracefully handle division by zero scenarios in user engagement calculations.

ROUND(SAFE_DIVIDE(SUM(event_count_without_session_start), COUNT(DISTINCT user_pseudo_id)), 2)

Key Technical Learnings

BigQuery Performance Optimization

Learned to leverage BigQuery's columnar storage and partitioning for efficient querying of large GA4 datasets. Using date-partitioned tables and proper WHERE clause filtering significantly reduced query costs and execution time.

GA4 Data Model Understanding

Gained deep insights into GA4's event-based data model, understanding the nuances of automatically collected events versus enhanced measurement events, and how to extract meaningful business metrics from raw event streams.

Advanced SQL Window Functions

Mastered complex window function applications for analytical use cases, particularly partitioned aggregations and ranking functions that enable sophisticated multi-dimensional analysis in a single query.
