Automated GTM ID Monitoring System

Implementing a serverless monitoring pipeline using Google Cloud Platform to ensure tracking accuracy across multiple domains

Project Duration: 2 weeks
Team Size: Individual project
Impact: 95% reduction in manual checks

The Challenge

Managing website tracking across multiple domains presented significant operational challenges. Our team was responsible for monitoring Google Tag Manager (GTM) implementations across 15+ websites, each critical for accurate data collection and marketing attribution.

Time-Consuming Manual Checks

Daily manual verification of GTM IDs across all domains was taking 2+ hours of analyst time

Human Error Risk

Manual processes were prone to oversight, potentially missing critical tracking failures

Communication Delays

No systematic way to alert the team when tracking issues were detected

Solution Architecture

I designed a serverless automation pipeline leveraging Google Cloud Platform services to create a reliable, scalable monitoring system.

Implementation Details

Core Python Script

The heart of the system is a Python function that performs HTTP requests to check for GTM container presence:

main.py 
import requests
import re
import json

# Configuration
SLACK_TOKEN = 'xoxb-your-slack-bot-token'
SLACK_CHANNEL = 'C1234567890'

# Website monitoring configuration
WEBSITES = {
    'https://example-site1.com': 'GTM-ABC123',
    'https://example-site2.com': 'GTM-DEF456',
    'https://example-site3.com': 'GTM-GHI789',
    # Add additional sites as needed
}

def check_gtm_presence():
    """
    Check for GTM container presence across configured websites
    Returns dict with status for each site
    """
    results = {}
    
    for url, gtm_id in WEBSITES.items():
        try:
            # Perform HTTP request with timeout
            response = requests.get(url, timeout=10, headers={
                'User-Agent': 'GTM-Monitor/1.0'
            })
            
            # Search for GTM ID in page source
            if re.search(f'{gtm_id}', response.text):
                results[url] = {
                    'status': 'success',
                    'message': 'GTM container found',
                    'gtm_id': gtm_id
                }
            else:
                results[url] = {
                    'status': 'warning',
                    'message': 'GTM container missing',
                    'gtm_id': gtm_id
                }
                
        except requests.RequestException as e:
            results[url] = {
                'status': 'error',
                'message': f'Request failed: {str(e)}',
                'gtm_id': gtm_id
            }
            
    return results

def format_slack_message(results):
    """
    Format monitoring results for Slack notification
    """
    message_blocks = [
        {
            "type": "header",
            "text": {
                "type": "plain_text",
                "text": "üîç Daily GTM Monitoring Report"
            }
        },
        {
            "type": "section",
            "text": {
                "type": "mrkdwn",
                "text": f"*Report Date:* {datetime.now().strftime('%Y-%m-%d %H:%M:%S')}"
            }
        }
    ]
    
    success_count = sum(1 for r in results.values() if r['status'] == 'success')
    total_count = len(results)
    
    # Summary section
    summary_text = f"*Summary:* {success_count}/{total_count} sites have GTM properly configured"
    if success_count == total_count:
        summary_text += " ‚úÖ"
    else:
        summary_text += " ‚ö†Ô∏è"
        
    message_blocks.append({
        "type": "section",
        "text": {
            "type": "mrkdwn",
            "text": summary_text
        }
    })
    
    # Detailed results
    for url, result in results.items():
        icon = "‚úÖ" if result['status'] == 'success' else "‚ùå" if result['status'] == 'error' else "‚ö†Ô∏è"
        
        message_blocks.append({
            "type": "section",
            "text": {
                "type": "mrkdwn",
                "text": f"{icon} *{url}*\n`{result['gtm_id']}` - {result['message']}"
            }
        })
    
    return {"blocks": message_blocks}

def post_to_slack(message_data):
    """
    Send formatted message to Slack channel
    """
    headers = {
        'Authorization': f'Bearer {SLACK_TOKEN}',
        'Content-Type': 'application/json'
    }
    
    payload = {
        'channel': SLACK_CHANNEL,
        **message_data
    }
    
    response = requests.post(
        'https://slack.com/api/chat.postMessage',
        json=payload,
        headers=headers
    )
    
    if not response.ok or not response.json().get('ok'):
        print(f'Failed to send message to Slack: {response.text}')
        return False
    
    return True

def main(event, context):
    """
    Cloud Function entry point triggered by Pub/Sub
    """
    print('Starting GTM monitoring check...')
    
    # Check all websites
    results = check_gtm_presence()
    
    # Format and send Slack notification
    message_data = format_slack_message(results)
    success = post_to_slack(message_data)
    
    if success:
        print('GTM monitoring report sent successfully')
    else:
        print('Failed to send monitoring report')
    
    return {'status': 'completed', 'results': results}

Deployment Process

1

Create Pub/Sub Topic

Establish the messaging infrastructure for triggering the function

gcloud pubsub topics create daily-gtm-check
2

Deploy Cloud Function

Upload and configure the Python script as a serverless function

gcloud functions deploy gtm-monitor \ --runtime python39 \ --trigger-topic daily-gtm-check \ --entry-point main \ --memory 256MB \ --timeout 540s
3

Configure Cloud Scheduler

Set up automated daily execution at 9 AM CET

gcloud scheduler jobs create pubsub daily-gtm-job \ --schedule="0 9 * * *" \ --topic=daily-gtm-check \ --message-body='{}' \ --time-zone="Europe/Warsaw"
4

Slack App Configuration

Create and configure Slack bot with appropriate permissions

  • Create new Slack app in your workspace
  • Add chat:write bot scope
  • Install app to workspace
  • Copy bot token for authentication

Key Learnings

Serverless Architecture Benefits

Using Cloud Functions eliminated the need for server management and provided automatic scaling. The pay-per-execution model made the solution extremely cost-effective for periodic monitoring tasks.

Error Handling & Reliability

Implementing comprehensive error handling and retry logic was crucial for reliable operation. The system gracefully handles network timeouts, invalid responses, and Slack API failures.

Team Communication Integration

Integrating monitoring results directly into existing team communication workflows (Slack) significantly improved response times and reduced the chance of missed issues.
